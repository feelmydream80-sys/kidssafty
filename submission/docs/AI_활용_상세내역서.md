# [공공데이터 활용 + AI 활용 상세내역서]

## 1. 프로젝트 개요

### 1.1 프로젝트명
**아이지킴이 AI (Child-Centric Smart Safety Guardian)**

### 1.2 프로젝트 목적
기존 "위험물(횡단보도, 사고지점) 중심"의 정적인 안전 분석에서 벗어나, **아동의 실시간 위치(GPS)를 기반으로 주변 위험 요소를 능동적으로 탐지하고 선제적 보호**하는 AI 시스템 구축

### 1.3 대상 사용자
- **아동**: 초중등학생 (7세~15세)
- **보호자**: 부모, 교사, 학교 관계자
- **지자체**: 스쿨존 안전 관리 담당자

### 2.1 활용 데이터 목록

| 번호 | 데이터명 | 제공기관 | 데이터포털 URL | 주요 항목 | 활용 목적 |
|------|---------|---------|---------------|----------|----------|
| 1 | 초중등학교 학교정보 | 한국교육학술정보원 | https://www.data.go.kr/data/15123572/fileData.do | 학교명, 주소, 위도/경도, 학생수, 학급수 | 아이 소속 학교 위치 기반 보호구역 설정 |

### 2.2 데이터 전처리 계획
- **좌표계 통일**: 모든 데이터를 WGS84 (GPS) 좌표계로 변환
- **시간 동기화**: 등하교 시간대(07-09시, 14-16시) 데이터 필터링
- **위험 가중치 부여**: 
  - 횡단보도: 신호등 있음(0.3), 없음(0.7)
  - 사고지점: 사고건수에 비례하여 0.1~1.0 스케일링
- **데이터 병합**: 아이 위치(기준점)로부터 반경 100m 내 모든 위험물 매핑

### 2.3 데이터 업데이트 주기
- 학교정보: 매년 1회 (신학기 전 업데이트)
- 횡단보도/보호구역: 분기별 1회 (지자체 갱신분 반영)
- 사고다발지역: 반기별 1회 (최신 사고 통계 반영)

## 3. AI 기술 활용 내역

### 3.1 AI 모델 1: 아이 이동 패턴 학습 (LSTM)

**모델 유형**: LSTM (Long Short-Term Memory) 기반 시계열 예측

**입력 데이터**:
- 시계열 GPS 궤적: (위도, 경도, 타임스탬프)
- 컨텍스트: 요일(one-hot), 시간대(등교/하교/기타), 날씨(클라우드 API)

**학습 방법**:
- 지도학습 (Supervised Learning)
- 정상 경로 데이터 (과거 1개월 GPS 로그) 학습
- 비정상 라벨링: 평소 이동 시간 대비 30% 이상 지연, 경로 이탈 등

**예상 성능**:
- 정상/비정상 분류 정확도: 85% 이상
- 경로 예측 오차 (Haversine distance): 50m 이내

### 3.2 AI 모델 2: 주변 위험 실시간 탐지 (CNN + 공간 매칭)

**모델 유형**: CNN (Convolutional Neural Network) + KD-Tree 공간 인덱싱

**입력 데이터**:
- 아이 현재 위치 (GPS)
- 공공데이터 위험물 좌표 (횡단보도, 사고지점) - 이미지 형태의 히트맵 변환

**처리 절차**:
1. KD-Tree로 아이 반경 100m 내 위험물 후보군 추출 (< 0.5초)
2. CNN 모델로 위험물 유형 분류 (횡단보도, 사고지점, 기타)
3. 위험도 산출: `위험도 = Σ(위험물가중치 × 거리감쇄함수)`

**거리감쇄함수**: 
```
f(d) = exp(-d/30)  (d: 아이와 위험물 간 거리, 단위: m)
```

**예상 성능**:
- 위험물 탐지 정확도: 90% 이상
- 응답 속도: 평균 0.8초 이내

### 3.3 AI 모델 3: 개인화 위험도 예측 (강화학습)

**모델 유형**: Deep Q-Network (DQN) 기반 강화학습

**환경 구성**:
- State: 아이 위치, 시간, 주변 위험물 수/종류, 과거 사고 이력
- Action: 알림 여부 (0: 무시, 1: 경고, 2: 긴급 알림)
- Reward: 
  - 아이가 안전하게 이동: +1
  - 위험 구역 진입 후 알림: -0.5
  - 선제적 알림으로 사고 예방: +2

**학습 데이터**: 시뮬레이션 환경에서 생성 (실제 사고 데이터는 소수이므로)

**목표**: 알림 피로도 최소화하면서 안전성 극대화

### 3.4 자연어 처리 (TTS 및 음성 안내)

**기술**: 
- Text-to-Speech (TTS): 네이버 클로바 등 국산 엔진 활용
- 아동 친화적 언어 스타일 변환 (예: "위험합니다" → "조심해! 차가 올 수 있어")

**음성 안내 시나리오**:
- 횡단보도 50m 전: "앞에 횡단보도가 있어요. 손을 들고 건너자!"
- 사고지점 진입: "여기는 사고가 많이 나는 곳이야. 무리하지 말고 천천히 가자."

## 4. 시스템 아키텍처

### 4.1 구성 요소
1. **데이터 수집 계층**: 공공데이터포털 API/파일 다운로드 (Python 스크립트)
2. **데이터 저장 계층**: 
   - 정적 데이터: PostgreSQL + PostGIS (공간 DB)
   - 실시간 데이터: Redis (아이 GPS 세션)
3. **AI 분석 계층**: Python (TensorFlow/PyTorch), REST API 제공
4. **앱 계층**: 
   - 아동용: PWA (Leaflet.js 지도, GPS 추적)
   - 보호자용: 반응형 웹 대시보드

### 4.2 데이터 흐름
```
[아동 스마트폰] --GPS--> [API 서버] --위치전송--> [AI 엔진]
                                                   |
[공공데이터] --주기적업데이트--> [데이터베이스] <---+
                                                   |
[AI 엔진] --위험도/알림--> [알림 서비스] --Push--> [아동 앱]
                                                   |
                                                   +--Push--> [보호자 앱]
```

아이지킴이 AI는 **공공데이터의 가치를 사람(아동) 중심으로 재해석**한 혁신적 프로젝트입니다. 단순히 데이터를 시각화하는 것을 넘어, AI가 아이의 이동을 실시간으로 학습하고 보호하는 **'움직이는 안전 지킴이'**를 구현합니다.
