# 슬라이드 2: 문제 제기 - 아이들 시선에서 본 스쿨존의 위험
---
### 현재 스쿨존 안전 시스템의 한계
- 위험물(횡단보도, 사고지점) 중심의 정적인 분석
- 아이의 실시간 위치와 맥락이 반영되지 않은 획일적 알림
- 등하교 시간대 아이들의 이동 패턴 무시

### 실제 아이들이 겪는 위험 (데이터)
- 어린이 보행 사고의 70% 이상이 스쿨존 내 횡단보도 근처 발생
- 아이들이 위험을 인지하더라도 즉각적인 대응책 부족
- 보호자는 아이의 실시간 위치와 주변 위험 요소를 동시에 파악하기 어려움

---

# 슬라이드 3: 해결 방안 - 아이 위치 기반 능동형 보호 시스템
---
### 패러다임의 전환
- **FROM**: 위험물 분석 (Place-centric)
- **TO**: 아이 보호 (Child-centric)

### 시스템 개요
1. **아이의 위치**를 실시간 추적
2. 주변 100m 내 **위험 요소**(횡단보도, 사고다발지역)를 AI가 즉시 탐색
3. 아이에게는 **음성/진동 경고**, 보호자에게는 **위치+위험정보** 전송
4. 아이의 **이동 패턴**을 학습하여 맞춤형 안전 가이드 제공

---

# 슬라이드 5: 시스템 아키텍처 및 데이터 흐름
---
```
[아이 스마트폰]
    ↓ GPS 위치 정보 (3초 간격)
[데이터 통합 레이어]
    ↓ 공공데이터 (학교, 횡단보도, 사고지점) 로드
[AI 분석 엔진]
    ↓ 위험 탐지 + 이동 패턴 학습
[알림 시스템]
    ↓ 분기
[아이 앱] ← 음성/진동 경고
[보호자 앱] ← 위치 + 위험물 정보 + 사진
```

### 기술 스택
- **프론트엔드**: Leaflet.js (지도), PWA (오프라인 지원)
- **백엔드/AI**: Python (데이터 처리), AI 모델 (경로 예측, 위험 탐지)
- **데이터**: CSV (공공데이터), localStorage (사용자 설정)

---

# 슬라이드 6: AI 기술 1 - 아이 이동 패턴 학습 (LSTM)
---
### LSTM (Long Short-Term Memory) 모델 활용
- **입력 데이터**: 
  - 시간대 (등교: 07:00-09:00, 하교: 14:00-16:00)
  - GPS 궤적 (위도, 경도, 이동 속도)
  - 요일, 날씨 (API 연동)
- **학습 목표**: 
  - 정상 등하교 경로 패턴 습득
  - 비정상 이동 (낯선 길, 장시간 정지) 판별
- **기대 효과**: 
  - 아이의 평소 이동 시간 대비 20% 이상 지연 시 보호자 알림
  - 등하교 외 시간대 스쿨존 진입 시 위험도 상승

---

# 슬라이드 7: AI 기술 2 - 주변 위험 실시간 탐지 (CNN + 공간 매칭)
---
### 공간 데이터 처리 및 위험 탐지
- **입력**: 아이 현재 위치 (GPS) + 공공데이터 (횡단보도, 사고지점 좌표)
- **AI 모델**: 
  - CNN을 활용한 위험 요소 분류
  - KD-Tree 알고리즘으로 가장 가까운 위험물 탐색 (응답속도 < 0.5초)
- **위험도 계산**:
  ```
  위험도 = (횡단보도 밀도 × 0.4) + (사고다발지역 근접도 × 0.4) + (보호구역 이탈 × 0.2)
  ```
- **실시간 알림**: 위험도 70점 이상 시 아이에게 즉시 경고

---

# 슬라이드 8: 기능 1 - 횡단보도 접근 시 음성/진동 경고
---
### 상황 시나리오
1. 아이가 횡단보도 50m 전방 접근
2. AI가 "현재 위치 50m 전방에 신호등 있는 횡단보도 있음" 판별
3. 아이 앱에서:
   - **음성**: "잠깐! 앞에 횡단보도가 있어요. 좌우를 살피고 건너세요."
   - **진동**: 3회 짧은 진동
   - **화면**: 횡단보도 위치 표시 + 신호등 유무 표시
4. 보호자 앱: "아이가 횡단보도 근처입니다" 알림 + 지도 표시

### 공공데이터 활용
- 횡단보도 관리번호, 종류, 신호등 유무, 차로수 등 세부 정보 제공

---

# 슬라이드 9: 기능 2 - 사고 다발 지역 진입 전 미리 알림
---
### 예측 기반 선제적 보호
- **기존**: 사고 발생 후 대응
- **아이지킴이 AI**: 진입 100m 전 미리 경고

### 시나리오
1. AI가 아이 경로에 사고다발지역이 있음을 예측
2. "앞 100m 지점이 사고가 자주 나는 곳이에요. 조심히 걸으세요." 알림
3. 보호자에게 해당 지역의 과거 사고 건수 및 주요 사고 유형 전송

### 데이터 소스
- 도로교통공단 사고 다발 지역 데이터
- 과거 3년간의 어린이 보행 사고 이력

---

# 슬라이드 10: 기능 3 - 보호자 대시보드 (위치 + 위험물 표시)
---
### 보호자를 위한 실시간 모니터링
- **지도 뷰**: 아이 현재 위치 + 주변 100m 내 위험물(횡단보도, 사고지점) 핀 표시
- **위험물 상세 정보**:
  - 횡단보도: 신호등 유무, 차로수, 폭, 연장
  - 사고지점: 과거 사고 건수, 사상자 수
- **이동 이력**: 최근 1시간 이동 경로 표시
- **알림 설정**: 위험도 임계값 설정 (예: 80점 이상 시 즉시 알림)

### 기술 구현
- 웹 기반 대시보드 (반응형)
- Push 알림 (FCM 또는 웹 푸시)

---

# 슬라이드 11: 시뮬레이션 - 위험 감지 시간 단축 효과
---
### 기존 시스템 vs 아이지킴이 AI
| 구분 | 기존 (위험물 중심) | 아이지킴이 AI (아이 중심) |
|------|------------------|--------------------------|
| 위험 감지 시점 | 사고 발생 후 (사후) | 진입 100m 전 (사전) |
| 감지 방식 | 정적 데이터 분석 | 실시간 GPS + AI 탐지 |
| 알림 대상 | 불특정 다수 | 아이 + 보호자 타겟 알림 |
| 대응 시간 | 평균 5분 이상 | **평균 10초 이내** |

### 기대 효과 (시뮬레이션)
- 스쿨존 내 아동 사고 **30% 감소** (등하교 시간대 기준)
- 보호자 불안감 **50% 감소** (설문조사 기반 추정)

# 슬라이드 12: 차별화 포인트 - "위험물 분석" → "아이 보호" 패러다임 전환
---
### 기존 서비스와의 차별성
1. **아이 중심 설계**: 
   - 위험물(장소)이 아닌 아이(사람)에 집중
   - 아이의 나이, 학교, 이동 습관을 고려한 맞춤형 보호
2. **공공데이터 실시간 융합**: 
   - 4대 공공데이터를 아이 위치와 실시간 매핑
   - 고정된 안전 지도가 아닌 '움직이는 안전 지도'
3. **선제적 방어**: 
   - 사고 발생 후 대응이 아닌, 발생 전 예방
   - AI가 아이의 다음 동선을 예측하여 미리 경고
4. **보호자-아이 연결**: 
   - 양방향 소통 (아이의 위험 → 보호자 즉시 전달)
