# Risk Prediction Module

## 현재 상태
- **비활성화 일자**: 2026-05-08
- **비활성화 사유**: 데이터 오염(leakage) 및 과적합 문제
- **현재 성능**: R²=0.998, Accuracy=1.000 (비현실적, 과적합)

## 알고리즘 개요
Random Forest / XGBoost를 사용한 아동 안전 사고 위험 구역 예측
- 입력: 격자점(100m 간격) + 주변 횡단보도/학교/사고다발지역 피처
- 출력: 위험 점수(0~1) 및 위험 구역 클러스터

## 🔴 치명적 문제점

### 1. 데이터 오염 (Data Leakage)
**위치**: `features.py` - `extract_features()` 함수

#### `acc_near_dist` (L89)
- **정의**: 사고다발지역까지의 최소 거리
- **문제**: 라벨(`is_danger`)이 "사고다발지역 50m 이내"로 정의됨
- **결과**: `acc_near_dist ≤ 50` = `is_danger=1` (100% 완벽 예측)
- **중요도**: 35% (가장 높음)

#### `acc_count_500m` (L88)
- **정의**: 500m 내 사고 건수 합계
- **문제**: 사고 구역 인근(라벨=1)에서 값이 높아 라벨 정보 직접 누출
- **중요도**: 약 15%

### 2. SMOTE 적용 오류
**위치**: `predict_risk_zones.py` L125, L132
- **현재**: 전체 데이터에 SMOTE 적용 → train/test 분할
- **문제**: 테스트 세트 정보가 학습에 유입됨
- **올바른 방법**: train 세트에만 SMOTE 적용

### 3. 샘플 부족
- **실제 양성 샘플**: 9개 (사고다발지역 50m 이내 격자점)
- **SMOTE 증강**: 9개 → 2044개 (합성 데이터)
- **문제**: 실제 데이터가 너무 적어 모델이 일반화되지 않음

### 4. 타겟 변수 한계
- **현재**: 이진 분류 (50m 이내 사고존 여부)
- **문제**: 양성 샘플 9개로는 의미있는 학습 불가
- **해결**: 연속형 타겟으로 변경 (500m 내 사고 건수 정규화)
